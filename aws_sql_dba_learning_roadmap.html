AWS SQL Server DBA learning roadmap for freshers, organized into 5 phases from cloud basics to advanced administration

Phase 1 — AWS Cloud Fundamentals ~3–4 weeks
Core AWS concepts
Regions, Availability Zones, VPCs
EC2 — instance types, AMIs, key pairs
IAM — users, roles, policies, least privilege
Security Groups, NACLs, subnets
S3 buckets — storage, versioning, lifecycle
AWS CLI basics — configure, run commands
Practical lab
Launch a Windows EC2 instance → connect via RDP → create an S3 bucket → upload/download files → create an IAM role and attach it to the EC2 instance.
Phase 2 — SQL Server on EC2 (Self-Managed) ~3–4 weeks
Install & configure SQL Server on EC2
Launch SQL Server AMI from AWS Marketplace
Configure SQL Server — memory, tempdb, filegroups
SQL Server logins, roles, permissions setup
Connect SSMS from local machine to EC2 instance
Backup to S3 using URL (native backup to S3)
EBS volumes — attach data disk, configure SQL data/log drives
Practical lab
Install SQL Server 2022 on EC2 → configure max memory and tempdb → create a test database → backup to S3 using BACKUP TO URL → restore from S3.
Phase 3 — AWS RDS for SQL Server (Managed) ~4–5 weeks
Provisioning & setup
Create RDS SQL Server instance (Console & CLI)
Editions — Express, Web, Standard, Enterprise
Parameter Groups & Option Groups
Subnet Groups, Security Groups for RDS
Backup, restore & maintenance
Automated backups — retention, backup windows
Manual snapshots — create, copy, share, restore
Point-in-time restore (PITR)
Maintenance windows — patching, minor upgrades
Native backup/restore — S3 import (.bak files)
Major version upgrades on RDS
High availability & scaling
Multi-AZ deployments — how automatic failover works
Read Replicas — create and promote
Storage scaling — autoscaling, manual resize
Vertical scaling — change instance class
Security on RDS
Encryption at rest (KMS) and in transit (SSL)
IAM database authentication
Secrets Manager for credential rotation
RDS audit logging — enabling SQL Server Audit
Practical lab
Create Multi-AZ RDS SQL Server → connect via SSMS → restore .bak from S3 → force a failover and observe the endpoint → take a snapshot → restore to point-in-time → enable encryption.
Phase 4 — Monitoring, Performance & Automation ~3–4 weeks
Monitoring tools
CloudWatch — metrics, alarms, dashboards for RDS
Performance Insights — top SQL, wait events
Enhanced Monitoring — OS-level metrics
RDS Events & SNS alerts
Performance tuning on AWS
DMVs inside RDS — what's accessible, what's not
Query Store — enable and use for regression analysis
Index analysis, missing index DMVs
Right-sizing instance classes for workloads
Automation
AWS CLI scripts for RDS management
PowerShell + AWS Tools for automation
Lambda functions to start/stop RDS on schedule
CloudFormation — deploy RDS stack as code
Practical lab
Enable Performance Insights on RDS → run a heavy query → identify it as top SQL → create a CloudWatch alarm for high CPU → write a Lambda function to take a snapshot every night at 2 AM.
Phase 5 — Migration, Security Hardening & Real-World Ops ~4 weeks
On-prem to AWS migration
Backup-restore migration (.bak to RDS via S3)
AWS DMS — replicate on-prem SQL to RDS live
SCT — Schema Conversion Tool for migrations
Pre/post migration validation checklists
Security hardening & compliance
VPC private subnets — no public RDS endpoints
SSL/TLS enforcement for all connections
AWS Config & GuardDuty for compliance monitoring
Secrets Manager — auto-rotate SQL credentials
Real-world operations
Incident response — RDS outage runbook
DR drill — restore from snapshot in another region
Documenting runbooks in SharePoint/Confluence
Change management — CAB process for AWS changes
Practical lab
Use DMS to replicate an on-prem (or EC2) SQL Server database to RDS in near real-time → validate row counts → cut over → restore the RDS snapshot to a different region as a DR drill.
